### 📝 2. 创建策略

#### 基础双均线策略

```bash
curl -X POST http://127.0.0.1:8108/api/strategies \
  -H "Content-Type: application/json" \
  -d '{
    "name": "黄金快慢均线策略",
    "dataset": "gold_daily",
    "status": "active",
    "spec": {
      "symbol": "XAUUSD",
      "direction": "long_only",
      "indicators": [
        {"name": "fast_ma", "type": "sma", "window": 12},
        {"name": "slow_ma", "type": "sma", "window": 36}
      ],
      "entry_rules": [
        {"left": "fast_ma", "op": "crosses_above", "right": "slow_ma"}
      ],
      "exit_rules": [
        {"left": "fast_ma", "op": "crosses_below", "right": "slow_ma"}
      ],
      "position_sizing": {
        "mode": "fixed_fraction",
        "risk_fraction": 0.1,
        "max_units": 100
      },
      "risk_limits": {
        "max_order_notional": 50000,
        "max_position_per_symbol": 100,
        "max_gross_exposure": 250000,
        "stop_loss_pct": 0.05,
        "take_profit_pct": 0.12
      }
    }
  }'
```

**支持的指标类型**:
- `sma`: 简单移动平均
- `ema`: 指数移动平均
- `rsi`: 相对强弱指标
- `macd`: MACD指标
- `momentum`: 动量指标
- `bollinger_bands`: 布林带
- `atr`: 平均真实波幅

**支持的操作符**:
- `>`: 大于
- `<`: 小于
- `>=`: 大于等于
- `<=`: 小于等于
- `crosses_above`: 向上穿越
- `crosses_below`: 向下穿越

## 常用功能示例

### 示例1: 创建完整的黄金交易策略

```bash
# 1. 创建策略
STRATEGY_RESPONSE=$(curl -X POST http://127.0.0.1:8108/api/strategies \
  -H "Content-Type: application/json" \
  -d '{
    "name": "黄金趋势跟踪策略",
    "dataset": "gold_daily",
    "status": "active",
    "spec": {
      "symbol": "XAUUSD",
      "direction": "long_only",
      "indicators": [
        {"name": "fast_ma", "type": "ema", "window": 12},
        {"name": "slow_ma", "type": "ema", "window": 26},
        {"name": "rsi", "type": "rsi", "window": 14}
      ],
      "entry_rules": [
        {"left": "fast_ma", "op": "crosses_above", "right": "slow_ma"},
        {"left": "rsi", "op": "<", "right": 70}
      ],
      "exit_rules": [
        {"left": "fast_ma", "op": "crosses_below", "right": "slow_ma"},
        {"left": "rsi", "op": ">", "right": 70}
      ],
      "position_sizing": {
        "mode": "fixed_fraction",
        "risk_fraction": 0.1,
        "max_units": 120
      },
      "risk_limits": {
        "max_order_notional": 50000,
        "max_position_per_symbol": 120,
        "max_gross_exposure": 250000,
        "stop_loss_pct": 0.05,
        "take_profit_pct": 0.12
      }
    }
  }')

# 获取策略ID
STRATEGY_ID=$(echo $STRATEGY_RESPONSE | jq -r '.strategy_id')

# 2. 运行回测
curl -X POST http://127.0.0.1:8108/api/backtest \
  -H "Content-Type: application/json" \
  -d "{
    \"strategy_id\": \"$STRATEGY_ID\",
    \"start_date\": \"2023-01-01\",
    \"end_date\": \"2023-12-31\"
  }"

# 3. 执行交易
curl -X POST http://127.0.0.1:8108/api/execute \
  -H "Content-Type: application/json" \
  -d "{
    \"strategy_id\": \"$STRATEGY_ID\",
    \"broker_mode\": \"paper\"
  }"
```

## 💡 最佳实践

### 1. 策略开发流程

```
创建策略 → 基础回测 → 高级回测 → 参数优化
     ↓
 MAE/MFE诊断 → 应用优化 → Paper交易
     ↓
   实盘监控 → 持续优化
```

### 2. 风险管理

- 单笔风险不超过1-2%
- 总持仓不超过总资金的50%
- 严格止损7-8%
- 避免过度交易

### 3. 数据更新

系统使用内置演示数据。实盘使用需要：
- 配置实时数据源（OpenBB, finshare等）
- 更新市场数据API
- 设置定时数据刷新任务
